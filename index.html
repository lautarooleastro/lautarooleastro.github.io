Description

This is a desktop application for managing a local ice cream factory. The complete system works as follows:

  • The user can manually add ice cream flavours, with their respective identification number and legal number (denoted as RNPA).
  • The system is connected via RS-232 to a scale, on which the user can place the recently produced ice cream buckets.
  • From there, all that remains is to select the flavor to log the bucket into the system. At the same time, it prints the respective label with all the information of the bucket.
  • The label has information on the flavor, weight, legal codes, production date, expiration date and a barcode (so the bucket can be scanned later).
  • The label is printed on a sticker and placed on the bucket.
  • This bucket is stored into the industrial fridge and is already recorded into the system so the user can manage their stock more efficiently.

The user can export this data to excel and generate reports and analysis.

When the user wants to send a batch of buckets to the sales branch, it is necessary to scan the desired buckets to ship and generate the dispatch list. All those selected buckets will now be tagged as "Shipped" and the dispatch list will be generated in PDF format.

In simple terms it is a website for buying and selling used cars, but…
What's special about it is that it uses a third-party database to complete the vehicle information.
This way, the seller only has to choose his car from a series of drop-down menus, which consists of:
Brand > Model > Generation > Series > Version > Equipment
The system can then automatically fill in all vehicle specifications, such as: number of cylinders, displacement, top speed, horsepower, physical dimensions and more.
Also, by having all the cars perfectly categorized, a better analysis of the data can be done, either to classify the cars or use the information to our advantage (for example, showing a particular user the cars of their favorite brand).
Flood Prevention
Website focused on flood awareness and prevention in the city of La Plata, as well as evacuation plans in case of emergency.
Stack: Python, Flask, MySQL, SQLAlchemy, Bootstrap 5.
Details

Flood Prevention

The flooding of the City of La Plata occurred in April 2013. With more than 400 mm of rain that accumulated in just 4 hours, the drainage systems were not enough... Unfortunately, human lives were lost and the damage caused in the city were huge.
This web page was part of the subject "Software Project". The main objective is to prevent and raise awareness about floods.
Modules:
  • Meeting points: To gather evacuees in case of emergency and provide help.
  • Evacuation routes: For efficient evacuation.
  • Flood zones: To inform residents so that they can be avoided.
  • Complaints: Users can report clogged sewers, garbage dumps or any incident that obstructs proper drainage.
It is a web page composed of two applications: one private for management and another public for access by end users (connected to the first through API endpoints).
